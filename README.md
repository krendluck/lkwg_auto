# 自动化助手 (Auto Assistant)

这是一个基于 Python 和 Tkinter 开发的桌面自动化辅助工具，主要包含**按键映射**和**图像识别与自动化按键**两大核心功能模块。此工具专为解决部分游戏应用（如《洛克王国：世界》等）鼠标按键疲劳及繁琐点击流程而定制优化，并使用 **Windows 硬件级输入 API** (`SendInput`) 防止游戏安全系统屏蔽。

## 🌟 主要功能

### 1. 键盘映射鼠标长按 (硬件级注入)
*   **功能描述**：将指定的键盘按键（或组合键）映射为鼠标左键操作，完美绕过游戏应用屏蔽。
*   **动作逻辑**：按下设定的热键时，模拟按下鼠标左键；松开热键时，模拟松开鼠标左键。
*   **适用场景**：需要频繁多次点击或超长时间保持鼠标左键按下的场景。通过使用键盘来有效降低鼠标微动损耗与手指疲劳。
*   **新增全局快捷键**：支持使用 `F6` 启动映射，`F7` 停止映射。

### 2. 图像识别与复杂状态监控 (基于 OCR & 特征匹配)
*   **功能描述**：根据指定窗口内的状态自动执行轮换、判定、点击等复杂操作逻辑。
*   **特性**：
    *   **窗口动态绑定 & 相对坐标**：自动检测当前可用窗口（默认包含并锁定“洛克王国：世界”）。通过透明遮罩方便地框选关注区域 (ROI)，获取后的坐标会自动转为相对此窗口的坐标，窗口位置如何移动均不影响识别。
    *   **特征图匹配 & 坐标识别**：通过 `pyautogui` 定位目标特征（默认 `status.png`），应对游戏中多种动态交互图标。
    *   **战斗状态识别与连带点击 (新增)**：支持配置额外的判定图（如 `esc.png`）及连带动作图（如 `yes.png`），在识别出处于战斗/特定状态时，自动追踪特定目标去执行“连带点击”。
    *   **OCR 数字识别逻辑**：自动截取状态特征图旁边的数字并通过 `Tesseract-OCR` 进行高精度二值化识别，动态捕获场上“单位（如精灵）”序号。
    *   **智能自动轮换**：
        *   识别场上未出场的单位（2号~6号），自动发送硬件级数字键盘按键轮换调度。
        *   所有单位轮换完毕时，依据“轮换间隔”挂机循环。
    *   **实时识别预览栏**：内置 6 个槽位的实时图像切割与识别结果展示 UI，可视化所有推算步骤。

## 🛠️ 环境依赖与安装

### 1. Python 库依赖
请确保你的环境安装了 Python 3.x。在终端运行以下命令安装必要的依赖库：

```bash
pip install keyboard pillow pyautogui pytesseract opencv-python numpy
```

### 2. Tesseract-OCR (必须安装)
图像识别模块中的数字状态提取极度依赖 Tesseract-OCR 引擎，必须手动安装并配置文件：
1. 下载并配置 **Tesseract-OCR** for Windows 客户端。
2. **⚠️ 路径配置**：代码中可能硬编码了 Tesseract 运行路径，请将其安装在以下默认目录，或在 `auto.py` 的源码中将其替换为你的实际安装路径：
   ```python
   pytesseract.pytesseract.tesseract_cmd = r'D:\Program Files\Tesseract-OCR\tesseract.exe'
   ```

## 🚀 使用说明

### 启动程序
在项目根目录下运行主程序（脚本启动时会自动请求 **系统管理员权限**，以确保底层 `SendInput` 可在任何窗口下生效）：
```bash
python auto.py
```

### 模块一：键盘映射鼠标
1. 在主界面上半部分的“一. 键盘映射鼠标”区域中：
2. 点击 **⏺ 录制** 按钮，然按下你想要绑定映射的键盘按键（例如 `c`, `alt+x` 等）。
3. 点击 **▶ 启动** （或按下 `F6`）。此时配置的热键即在后台隐式化身为鼠标左键。
4. 点击 **⏹ 停止** （或按下 `F7`）取消一切映射。

### 模块二：图像识别点击
1. 在界面下半部分的“二. 图像识别点击”区域：
2. **选择目标窗口**：在下拉列表中选择欲驻留的目标独立窗口（如点击“刷新”可获取最新窗口信息）。
3. **框选关注范围**：点击“框选范围”，屏幕将变暗，拖拽鼠标框出你需要专门监控的区域（缩小区域能极大幅度提升识别性能）。
4. **特征图准备**：准备三张素材截图，放在脚本目录下：
   *   **状态标志** (如 `status.png`)：默认目标扫描图。
   *   *(选填)* **战斗判定** (如 `esc.png`)：配置相应的搜索坐标系 ROI，用于特殊判定。
   *   *(选填)* **连带点击** (如 `yes.png`)：配置检测到战斗及轮换时自动顺延点击的次要识别图及 ROI。
5. **设置参数**：按需调整“常规间隔(秒)”和“轮换间隔(秒)”。
6. 观察界面下方的“**实时识别预览**”以确保截屏被系统合法读取。
7. 点击 **▶ 开始识别** （或按下 `F8`）。

## ⌨️ 全局快捷键 (高优先度)

在程序执行期间，无论此工具是否隐藏在后台，以下硬件级快捷键全局并始终生效：

*   **`F6`** : 开启按键映射 (化键盘为鼠标)
*   **`F7`** : 停止按键映射
*   **`F8`** : 开始图像识别 & 自动轮换
*   **`F9`** : 停止图像识别模块
*   **`F10`**: 临时 暂停/恢复 自动轮换和按键发送逻辑

## ⚠️ 注意事项与免责声明

1. **UAC以管理员运行**：程序使用了 `SendInput` 构造底层硬件指令向量，默认会自动弹出 UAC 以**管理员身份**重启。若未弹窗，请手动使用管理员权限打开终端/IDE。
2. **多显示器与分辨率缩放 (DPI)**：若系统存在显示缩放（如 125%、150%），坐标定位可能产生漂移。建议在 python.exe 的属性中勾选“替代高DPI缩放行为（应用程序执行）”。
3. **OCR 误判处理**：游戏内多变背景、抗锯齿字体对 OCR 是极大挑战。程序使用中值滤波、颜色范围提取进行高强度纠错清洗（部分如将 `z` 纠正为 `2`、`S` 为 `5`）。
4. **免责声明**：本开源研究型工具仅供学习 Windows API, CV(计算机视觉) 及自动化流程测试探讨使用。请勿用于违规的“自动化游戏外挂”操作，否则造成的账号异常或损失请使用者自行承担。
