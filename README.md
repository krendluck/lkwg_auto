# 自动化助手 (Auto Assistant)

这是一个基于 Python 和 Tkinter 开发的桌面自动化辅助工具，主要包含**按键映射**和**图像识别与自动化按键**两大功能模块。特别针对《洛克王国：世界》等游戏或应用场景进行了定制优化。

## 🌟 主要功能

### 1. 键盘映射鼠标长按
*   **功能描述**：将指定的键盘按键（或组合键）映射为鼠标左键。
*   **动作逻辑**：按下设定的键盘按键时，模拟按下鼠标左键；松开键盘按键时，模拟松开鼠标左键。
*   **适用场景**：需要频繁或长时间点击/按住鼠标左键的场景，用键盘代替以减轻鼠标损耗和手指疲劳。

### 2. 图像识别与状态自动化 (基于 OCR)
*   **功能描述**：自动监控指定窗口的特定区域，寻找特征图标，并通过 OCR 识别图标旁的数字，从而自动进行特定的按键操作。
*   **特性**：
    *   **窗口绑定**：自动获取系统当前可用窗口，支持绑定到特定窗口（默认包含“洛克王国：世界”）。
    *   **相对坐标框选**：支持在屏幕上框选识别区域，并自动转化为相对该窗口的坐标，窗口移动不影响识别。
    *   **特征图匹配**：通过 `pyautogui` 寻找特征图（默认寻找 `status.png`）。
    *   **OCR 数字识别**：截取特征图旁边的数字并通过 `Tesseract-OCR` 进行识别，判断当前已出场“单位（精灵/角色）”序号。
    *   **自动轮换逻辑**：
        *   自动寻找 `2` 到 `6` 号位中**未出场**的最小序号，并模拟按下对应的数字键。
        *   如果全部出场，则按照设定的“轮换间隔”循环按键 `2~6`。

## 🛠️ 环境依赖与安装

### 1. Python 库依赖
请确保你的环境安装了 Python 3.x，然后在终端运行以下命令安装必要的依赖库：

```bash
pip install keyboard pillow pyautogui pytesseract opencv-python numpy
```

### 2. Tesseract-OCR (必须安装)
图像识别模块中的数字提取依赖于 Tesseract-OCR 引擎，你需要手动安装它：
1. 下载并安装 **Tesseract-OCR** for Windows。
2. **⚠️ 路径配置**：代码中硬编码了 Tesseract 的可执行文件路径，请确保它安装在以下路径，或在 `auto.py` 中修改对应的代码路径：
   ```python
   pytesseract.pytesseract.tesseract_cmd = r'D:\Program Files\Tesseract-OCR\tesseract.exe'
   ```

## 🚀 使用说明

### 启动程序
在项目根目录下运行主程序：
```bash
python auto.py
```

### Tab 1：键盘映射鼠标
1. 切换到“键盘映射鼠标”标签页。
2. 点击 **⏺ 录制** 按钮，然后按下你想要绑定的键盘按键（例如 `a`, `alt+x` 等）。
3. 点击 **▶ 启动**。此时你配置的按键就已经映射为鼠标左键了。
4. 点击 **⏹ 停止** 取消映射。

### Tab 2：图像识别点击
1. 切换到“图像识别点击”标签页。
2. **选择目标窗口**：在下拉列表中选择你要抓取的窗口（如“洛克王国：世界”）。
3. **框选识别区域**：点击“框选范围”，在屏幕上拖拽鼠标框出你需要监控的区域（最好刚好包含状态图标动态出现的范围）。
4. **特征图准备**：准备一张需要识别的小截图，命名为 `status.png` 放在程序同级目录，或点击“浏览...”选择其他图片。
5. **设置参数**：按需调整检测间隔和轮换间隔。
6. 点击 **▶ 开始识别** （或按下全局快捷键 `F8`）。

## ⌨️ 全局快捷键

在程序运行期间，无论程序是否在后台，以下快捷键均有效（主要用于图像识别功能）：

*   **`F8`** : 开始图像识别
*   **`F9`** : 停止图像识别
*   **`F10`**: 暂停/恢复 自动轮换操作

## ⚠️ 注意事项与免责声明

1. **管理员权限**：部分游戏或应用可能会屏蔽普通权限下的虚拟按键输入，建议以**管理员身份**运行终端和 Python 脚本。
2. **按键冲突**：请确保绑定的热键不会与系统中其他软件的全局热键发生冲突。
3. **OCR 准确率**：OCR 对于游戏内的特殊字体识别率可能有限，目前代码中内置了基于二值化处理和简单的字符替换纠错（如 `z`->`2`, `S`->`5`）。如果识别仍然不准，建议检查截图区域（`num_box_x`, `num_box_y` 等参数）是否对齐，并在 `debug` 日志中排查。
